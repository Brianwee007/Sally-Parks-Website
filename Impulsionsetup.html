7. Data Storage Note

Current Architecture
All horse data is stored in localStorage under the key parkwood_horses_v2 in the visitor's browser. This means:

✓ No server required — works entirely in GHL custom pages
✓ Admin edits appear instantly on horses/detail pages (same browser)
✗ Data is NOT shared across different browsers or devices
✗ Different visitors see different horses depending on their browser

Recommended upgrade path: Connect to Airtable (free) or a JSON API for true shared data. Advise client to upgrade once they confirm the system works for them.
